client upgrade request

GET {path} HTTP/1.1
Host: {host}
Connection: Upgrade, HTTP2-Settings
Upgrade: h2c
HTTP2-Settings: {base64url encoding of HTTP/2 SETTINGS payload}

server upgrade response (not supported)

ignore upgrade header (i.e. no specialcasing needed) 
HTTP/1.1 200 OK
Content-Length: {length}
Content-Type: {type}

{body}

server upgrade response (supported)

HTTP/1.1 101 Switching Protocols
Connection: Upgrade
Upgrade: h2c

client connection preface

c string literal: 
PRI * HTTP/2.0\r\n\r\nSM\r\n\r\n
or equivalent hex: 
505249202a20485454502f322e300d0a0d0a534d0d0a0d0a
followed by SETTINGS frame which may be empty.

SETTINGS frame

(if malformed, respond with connection error PROTOCOL_ERROR)
(if length % 6 != 0, respond with connection error FRAME_SIZE_ERROR)
type: 0x4
stream identifier: 0x0
(if not 0x0, respond with connection error PROTOCOL_ERROR)
flag: ack (0x1)

format

16 Identifier
32 Value
enum Identifier
0x1 SETTINGS_HEADER_TABLE_SIZE      type
0x2 SETTINGS_ENABLE_PUSH
0x3 SETTINGS_MAX_CONCURRENT_STREAMS
0x4 SETTINGS_INITIAL_WINDOW_SIZE
0x5 SETTINGS_MAX_FRAME_SIZE
0x6 SETTINGS_MAX_HEADER_LIST_SIZE
end
